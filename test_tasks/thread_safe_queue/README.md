## Техническое задание

### Реализовать шаблонную циклическую очередь

Необходимо реализовать шаблонный класс циклической очереди, удовлетворяющий следующим требованиям:

**Требования**
- Очередь должна быть безопасна для одновременной записи и чтения из двух потоков.
- Один поток может только читать из очереди, другой — только писать.
- Не использовать механизмы взаимной блокировки потоков.
- Память для хранения данных должна аллоцироваться статически внутри очереди.
- Шаблонные параметры: T - тип данных, CAPACITY - максимальный размер очереди.

**Требования к типу T**
- Наличие конструктора копирования.
- Наличие оператора присваивания.

**Прочее**
- Уничтожение объекта должно происходить сразу после его извлечения из очереди.
- Очередь должна поддерживать конструктор копирования и оператор присваивания.

## Реализация

- **Lock-free SPSC**: атомарные индексы `head_` и `tail_`, монотонно возрастающие.
- **Поведение при переполнении**: `Push` возвращает `false`, данные не перезаписываются.
- **Память**: `std::aligned_storage_t` для статической аллокации.
- **Тесты**: юнит-тесты + многопоточная проверка на 100 000 элементов.
